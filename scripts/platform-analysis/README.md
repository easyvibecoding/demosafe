# Platform Analysis Scripts

Developer maintenance tools for keeping `capture-patterns.ts` up-to-date with platform UI changes.

## How it works

These are **not standalone scripts**. They are reference files for AI Agents (like Claude Code) that use Chrome MCP tools to analyze platform DOM structures.

## Usage

Ask your AI Agent:

```
"Please analyze the OpenAI API keys page DOM structure.
Navigate to platform.openai.com/api-keys, run the analyze-dom.js script,
and compare the results with capture-patterns.ts."
```

The Agent will:
1. Navigate to the target page using `mcp__claude-in-chrome__navigate`
2. Execute `analyze-dom.js` via `mcp__claude-in-chrome__javascript_tool`
3. Compare results with existing patterns in `capture-patterns.ts`
4. Suggest updates if selectors have changed

## Files

- `analyze-dom.js` — Universal DOM analysis script (executed by Agent on target pages)
- `reports/` — Analysis reports (gitignored, generated by Agent)

## Reports

Reports are saved as JSON in `reports/` and are gitignored. They contain DOM structure info (tag names, classes, element paths) but NOT actual API key values.
