# 狀態機與操作模式

## 雙模式架構

整個產品運作於兩個基本模式：

| 模式 | 行為 | 視覺指示 |
|------|------|---------|
| **管理模式** | 可在設定頁查看 Key、組織、匯入。用於展示前準備。 | Menu Bar 圖示：預設色（單色） |
| **展示模式** | 所有 Key 全面遮蔽。貼上操作將真實 Key 送入剪貼簿但顯示遮蔽版本。錄影安全。 | Menu Bar 圖示：**紅/橙色盾牌** |

切換方式：`⌃⌥⌘D`（Control + Option + Command + D）。三個修飾鍵防止意外觸發。

---

## 情境模式 (Context Modes)

情境模式是預設的配置組合，控制遮蔽行為、剪貼簿策略和啟用的 Key 集合。

| 情境 | 遮蔽等級 | 剪貼簿策略 | 使用場景 |
|------|---------|-----------|---------|
| **直播中** | 所有表面全面遮蔽 | 30 秒後自動清除 | 產品發佈、現場展示 |
| **教學錄影** | 所有表面全面遮蔽 | 10 秒後自動清除 | YouTube、教學內容 |
| **內部展示** | 部分遮蔽（顯示前綴 + 末 4 碼） | 正常剪貼簿行為 | 團隊會議、內部簡報 |
| **開發中** | 遮蔽停用 | 正常剪貼簿行為 | 個人開發、除錯 |

使用者可建立自訂情境模式並綁定到特定快捷鍵。

### ContextMode 資料結構

| 屬性 | 型別 | 說明 |
|------|------|------|
| `maskingLevel` | Enum | `.full` \| `.partial` \| `.off` |
| `clipboardTTL` | Int? | 自動清除超時秒數 |
| `activeServiceIds` | [UUID]? | 此情境中啟用遮蔽的服務 |
| `shortcutKey` | String? | 啟用此情境的快捷鍵 |

> MVP 注意：情境預設為固定值（Livestream、Development），動態情境切換保留給未來版本。
